## Getting Started



### Run the code


1. Before run the application, cd to server directory to install the node_modules

  ```sh
  npm install 
  ```
2. Create an .env.local file into your server/client directory. I'll send the invitation to env github repo, you will need to copy/paste to your env.local files
```sh
   touch .evn.local
   ```
   
3. To start client/server (client has to be run on localhost:3000)
```sh
   npm start
   ```
4. Backend(server) structure

In our server code, I've organized everything into specific folders for clarity :

  &nbsp;&nbsp;<b>Controllers</b>: Incoming requests and outgoing responses. It's the bridge between the user and our server logic.

  &nbsp;&nbsp;<b>Services</b>: all main functions and business logic.

  &nbsp;&nbsp;<b>Models</b>: Even though we use Turso and don't need traditional models, I keep our database table creation SQL here. It helps everyone understand our database structure.

  &nbsp;&nbsp;<b>Routes</b>: This folder contains all our API endpoints. It's how we define the paths and actions the server handles.

  &nbsp;&nbsp;<b>index.js</b>: This is the starting point of the server
  
5. Frontend(client) structure

I organized it as the way that I usually do, but if you feel it need to be changed, feel free to adjust it

Under src folder:

  &nbsp;&nbsp;<b>assets</b>

  &nbsp;&nbsp;<b>components</b>

  &nbsp;&nbsp;<b>pages</b>

  &nbsp;&nbsp;<b>App.js</b>

6. To connect to Turso DB, you need to create a Turso account, and  I need to add your account to the project database, then you can access to the database through Turso Shell CLI
